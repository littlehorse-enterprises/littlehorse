# Littlehorse 1.1 


LittleHorse 1.1 introduces comprehensive workflow and task metrics, native array support, new variable operations (POW, SIZE_OF),
and significant observability improvements. The Dashboard gains metrics visualization charts, a datetime picker, and server version display.
All SDKs receive feature parity updates including struct flexibility enhancements and the new POW operation support. Native array support is initially available in the Java and .NET SDKs, with broader SDK coverage coming in the next minor version.


### Server
- Introduced time-windowed workflow and task metrics collection with partitioned and globally aggregated results, queryable via the new ListWfMetrics API and lhctl. #2071
- Added tenant-level aggregated metrics, enabling queries across all workflows or tasks within a tenant without specifying a name. #2171
- Added GetMetricWindow and SearchWfMetricWindow RPCs, allowing metric windows to be retrieved by ID or searched by workflow name and time range. #2204
- Added Micrometer counter metrics for command processor observability (lh.commands.processed, lh.subcommands.processed), exposed to Prometheus. #2242
- Introduced native LH Arrays as a first-class variable type with support for EXTEND, REMOVE_IF_PRESENT, REMOVE_INDEX mutations and IN / NOT_IN conditions. #2179
- Added SIZE_OF unary operation for computing the length of strings, arrays, and JSON arrays. #2230
- Added POW variable mutation type for exponential operations on numeric values. #2243
- Added is_nullable flag to StructFieldDef, allowing individual struct fields to be explicitly marked as nullable. #2215
- Implemented the Quota proposal, enabling server-side resource quota enforcement per tenant. #2141
- ⚠️ Forbidden JSON types (JSON_OBJ, JSON_ARR) inside StructDefs — use native Struct or Array types instead. #2262
- Added EXTRA_CLASSPATH environment variable support on server startup scripts, enabling runtime injection of external JARs (e.g., for AWS MSK IAM auth) without rebuilding the image. #2135

### Dashboard
- Added workflow metrics visualization panel on the WfSpec page, with interactive line/pie charts, configurable time ranges, bucket sizes, and auto-refresh. #2199
- Added a date-time picker component for timestamp inputs, supporting nanosecond precision via calendar + time field or Unix epoch milliseconds. #2201
- Added ExitNodeRun component with return output display in workflow run views. #2150
- Added WfSpec versioning UI, letting users view and navigate between workflow specification versions. #2157
- Dashboard now displays the connected server version. #2167

### SDK — Java
- Allows defining a Struct variable in a WfSpec by name or version without requiring a local reference to the StructDef class. #2207
- Added support for Type Adapters on WorkflowImpl#registerWfSpec. #2137
- Allows interaction with raw InlineStruct values directly. #2132
- Clients can now supply a custom worker task method resolver, giving greater control over task dispatch. #2193
- Exposes required child WfSpec names on the Workflow object. #2231

### SDK — Go
- Added WorkflowThread#RunWf() for running child workflows within a thread. #2134
- Allows defining a Struct variable by name or explicit version without a local StructDef reference. #2221
- Added POW operation for numeric variable mutations. #2245
- ⚠️ Forbidden JSON types in StructDefs — use native types instead. #2261

### SDK — .NET
- Added native LH Arrays support: declare array variables, use them in task methods via isLHArray annotation, and leverage arrays in StructDefs. #2259
- Allows defining a Struct variable by name or explicit version without a local StructDef reference. #2220
- Added POW operation for numeric variable mutations. #2244

### SDK — Python
- Added optional struct_def_version parameter to declare_struct, allowing versioned struct variable declarations without a class reference. #2222
- Added POW operation for numeric variable mutations. #2246
- ⚠️ Forbidden JSON types in StructDefs — use native types instead. #2260

### SDK — JavaScript / TypeScript
- Added mTLS support via LHC_CLIENT_CERT and LHC_CLIENT_KEY environment variables. #2177

### Test Utils
- Added @WithStructDefs annotation for ordered StructDef registration before E2E tests. #2194
- Enabled StructDef schema evolution (FULLY_COMPATIBLE_SCHEMA_UPDATES) and prevented duplicate registrations when multiple tests reference the same StructDef. #2212
