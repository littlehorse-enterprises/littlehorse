# LittleHorse 1.2

LittleHorse 1.2 introduces significant new workflow capabilities and a set of stability and correctness improvements. This release includes two experimental features that are planned to become generally available in the next version: **live migration** of running workflows to newer `WfSpec` versions, 
and **background bulk deletion of WfRuns** through the new `BulkJob` system. Additional `WfSpec` enhancements include array concatenation and a new native support for `Map` types. The dashboard has received a substantial improvements, redesigned `WfSpec` and `TaskDef` detail pages, new tables for `WfRun`s and scheduled runs, and quick jump-to-run search. This release also expands `StructDef` support across the SDKs and `lhctl`, alongside numerous stability, correctness, and access-control fixes.

## Features 

###  Server

- Added native `Map` support as a first-class variable type, enabling richer data modeling within `WfSpec`s. #2368
- `WfSpec` can now concatenate arrays using the `EXTEND` operation. #2385
- Added counted tags to the existing tag architecture (proposal 021), enabling efficient counts of tagged objects. #2265
- You can now count the total number of NodeRuns within a tenant, giving better visibility into tenant usage. #2304
- The server now rejects JSON types inside `StructDef`s. #2262
- EXPERIMENTAL: `WfRun` instances can now be migrated to a newer version of their `WfSpec`. #2336
- EXPERIMENTAL: Introduced a first-class BulkJob system, starting with background bulk deletion of WfRuns. Deletions run sharded per partition in the background without blocking Kafka Streams on large transactions, and jobs can be searched by status and cleaned up once finished. #2370
- Added a new `InlineStructDef` type to TypeDefinition, laying the groundwork for defining inline `StructDef`s within a ``WfSpec``. #2257

### Dashboard

- The `WfSpec` detail page has been redesigned and now surfaces storage usage, making it easier to inspect `WfSpec`s and understand their footprint on storage. #2301
- The `TaskDef` detail page and search experience have been improved for faster navigation and clearer insights into `TaskDef`s. #2302
- Added dedicated tables for `WfRun`s and scheduled runs, improving visibility into running and upcoming `WfRun`s. #2247
- A new header link lets you jump directly to a `WfRun` by ID from anywhere in the dashboard. #2312
- The dashboard now remembers your preferred items-per-load setting between sessions. #2253
- Route paths have been centralized for more consistent and reliable in-app navigation. #2254


### SDKs

- JavaScript/TypeScript SDK: switched to a new protobuf-to-TypeScript compilation plugin, changing the generated types across the SDK and dashboard. #2373
- Java SDK: `LHStructDef` annotations now support `StructDef` placeholders, simplifying how `StructDef` types are declared. #2389
- Java SDK: `StructDef` placeholders are now applied when ingesting `PollTask` values, ensuring struct values are correctly populated for task workers. #2404
- .NET SDK: added support for Array variables. #2259
- Added a pow (exponent) operation for numeric computations in the Python, Go, and .NET SDKs. #2246, #2245, #2244
- Go and Python SDKs now forbid JSON types in `StructDef`s, keeping `StructDef`s typed. #2261, #2260


### Dependencies

Littlehorse 1.2 now uses Kafka Streams 4.3.1

### `lhctl`

- lhctl now supports Map, Array, and Struct types, letting you work with complex variable types directly from the command line. #2384


## Bug Fixes

- Fixed nondeterministic WfSpec revision bumps: semantically identical `WfSpec` are now compared using canonicalized protobufs, so re-submitting the same spec no longer creates unnecessary new revisions. #2381
- Fixed a RocksDB iterator leak by properly closing iterators on the read-only base store, improving server stability and memory usage. #2364
- Resolved a race condition when updating metadata objects. #2291
- ACL actions are now validated against specific resources, tightening access-control correctness. #2367
- Pinned the authoritative element type on nested arrays and maps so their inner types are resolved correctly. #2398
- Fixed handling of arrays containing non-primitive types. #2250
- Gated BigDecimal conversion during JSONPath evaluation, preventing incorrect numeric handling. #2391
- Task timeout scheduling now sets the correct attempt number, ensuring timeouts apply to the right task attempt. #2340
- Prevented orphaned ThreadRuns when a workflow exceeds the maximum ThreadRun limit, avoiding inconsistent run state.
